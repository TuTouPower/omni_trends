## 一、项目简介

**NewNow** 是一个开源的实时新闻聚合平台，将多个信息源（微博热搜、知乎、36氪、Hacker News、Product Hunt 等）聚合到一个页面展示。支持 Cloudflare Pages / Docker / 本地部署。

## 四、问题根因分析

### 1. 项目缺乏维护

从 Issues 的时间跨度看，大量 Issue 从 2025 年中开始积累，作者 **ourongxing** 几乎没有回复或处理。129 个 Open Issue 中几乎没有被标记 label 或 assignee，说明项目处于**半停滞状态**。

### 2. 反爬策略跟进不足

NewNow 的核心依赖——各信息源的网页结构——在不断变化。网站的改版、验证码升级、API 限制等都会导致数据源失效。这需要持续的维护跟进，但目前完全缺失。

### 3. 部署体验碎片化

项目支持多种部署方式，但每种都有未解决的问题。用户在 Cloudflare Pages、Docker、Nginx 三条路径上都有大量报错，且缺乏统一的排障文档。

### 4. 缺乏社区贡献机制

项目没有 Issue 模板（#150 提到），也没有 Contributing 指引。大量 Issue 是低质量的单行描述（如 "新闻" #298、"自定义" #187），作者和社区之间缺乏有效沟通渠道。

## 五、如果你现在要用，需要注意什么

### 能正常工作的部分

- **Docker 部署**：基本可用，但部分数据源可能获取失败
- **Cloudflare Pages**：可以部署成功，但 OAuth 登录可能需要额外配置
- **静态数据源**：基于 RSS/公开 API 的源（如少数通过 API 获取的源）通常正常

### 需要自己解决的问题

1. **反爬失效的源**：需要自己 fork 并修复对应的源适配代码
2. **部署问题**：参考 #131（26 条评论的 Cloudflare 部署成功记录）和社区方案
3. **自定义需求**：目前不支持自定义数据源/排序，需要改代码
4. **MCP 集成**：当前 MCP Server 存在问题（#314, #263），不建议依赖

### 替代方案建议

如果你只是需要一个可用的新闻聚合：
- **先 fork 仓库**，修复你能修复的源
- **只用 Docker 部署**，避免 Cloudflare 的各种坑
- **不依赖 OAuth 登录**，目前该功能不可靠
- **接受部分源缺失**的现实，或自己写数据源适配器

## 八、总结

NewNow 是一个**想法很好但维护不足**的项目。19.5k star 说明需求真实存在，但 129 个 Open Issues 无人处理（尤其是数据源大面积失效）意味着**直接拿来用会踩很多坑**。

如果你现在要用，建议：
1. **Fork 后自行维护**，这是最现实的路径
2. **只保留你能验证正常工作的数据源**，禁用失效的
3. **Docker 部署**最可控，Cloudflare Pages 问题太多
4. **不要依赖 OAuth 登录和 MCP 功能**
5. 关注 #131（Cloudflare 部署经验帖）和 #252（OAuth 配置解决方案）中的社区讨论
